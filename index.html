vert.x

Effortless asynchronous application development for the modern web and enterprise

Polyglot


Write your application components in JavaScript, Ruby or Java. Or mix and match several programming languages in a single application.

Simplicity


...without being simplistic. Create real, scalable applications in just a few lines of code. No boilerplate or sprawling xml config.


Scalability


Natural scalability using messaging passing and immutable shared data to efficiently utilise your server cores.


Concurrency


Super-simple concurrency model frees you from worrying about locks and race conditions.



Features


  • Distributed Event Bus which spans the server and even penetrates into client side JavaScript.

  • Out of the box event bus modules including persistor, mailer, authentication manager, and work queues.

  • TCP/SSL servers and clients

  • HTTP/HTTPS servers and clients

  • WebSockets support

  • SockJS support

  • Extensive documentation

  • Open Source. Licenced under the ASL 2.0

  • Embeddable. If you don't want to run the whole vert.x platform, you can use vert.x embedded in your own pre-existing Spring Framework or other application.

  • Many other features. See the documentation for full details.

Here's a highly scalable web server. It serves files from the webroot directory.

(Click on the tab below for your preferred language)

  • JavaScript
  • Ruby
  • Java

Save the following in server.js

load('vertx.js')

new vertx.HttpServer().requestHandler(function(req) {
    var file = req.path === '/' ? 'index.html' : req.path;
    req.response.sendFile('webroot/' + file);
}).listen(8080)

Run it with:

vertx run server.js

Have lots of cores on your server and want to use them all?

Just spin up more instances:

vertx run server.js -instances 32

Vert.x will do the magic to ensure the requests are distributed amongst the instances:

Save the following in server.rb

require "vertx"
include Vertx

HttpServer.new.request_handler do |req|
  file = req.uri == "/" ? "index.html" : req.uri
  req.response.send_file('webroot/' << file)
end.listen(8080)

Run it with:

vertx run server.rb

Save the following in Server.java

import org.vertx.java.core.*;
import org.vertx.java.core.http.*;

public class Server implements Verticle {

  public void start() {
    new HttpServer().requestHandler(new Handler<HttpServerRequest>() {
      public void handle(HttpServerRequest req) {
        String file = req.path.equals("/") ? "index.html" : req.path;
        req.response.sendFile("webroot/" + file);
      }
    }).listen(8080);
  }

  public void stop() {
  }
}

Compile it:

javac Server.java -cp $VERTX_HOME/lib/jars/vert.x.jar

Run it with:

vertx run Server

Have lots of cores on your server and want to use them all?

Just spin up more instances:

vertx run Server -instances 32
